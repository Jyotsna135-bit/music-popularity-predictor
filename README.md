# 🎵 HitScore — Music Popularity Prediction Platform

Predicts a Spotify-style popularity score (0–100) for a track using either
its audio features or a raw MP3/WAV upload, and explains *why* with SHAP.

**[Live demo →](#)** *(add your Streamlit Cloud link here once deployed)*

## Overview

HitScore is an end-to-end ML project: a training pipeline that engineers
features and compares/tunes gradient-boosted models, and a Streamlit app
that serves predictions with explainability and actionable feedback.

- Upload an MP3/WAV/FLAC → features are extracted automatically with `librosa`
  (tempo, energy, danceability, valence, bass, spectral centroid, MFCCs, ...)
- Or tune audio features manually with sliders
- Get a popularity score, a radar chart of the track's "DNA", a SHAP
  waterfall of what drove the score, and concrete suggestions to improve it

## Model

| Metric | Value |
|---|---|
| Model | XGBoost (Optuna-tuned) |
| Test R² | ~0.81 |
| Test MAE | ~4.9 |
| Test RMSE | ~9.6 |
| Trained on | 114k tracks |

Trained with a 5-fold cross-validated comparison against LightGBM, Random
Forest, and a dummy baseline, then hyperparameter-tuned with Optuna (50
trials). See `notebook/music_popularity_advanced.ipynb` for the full
pipeline, model comparison charts, and SHAP analysis.

## Getting started

```bash
git clone https://github.com/<your-username>/hitscore.git
cd hitscore
pip install -r requirements.txt

# 1. Run the notebook once to train the model and populate models/
jupyter notebook notebook/music_popularity_advanced.ipynb

# 2. Launch the app
streamlit run app.py
```

The app expects `models/best_model.pkl`, `models/scaler.pkl`, and
`models/features.json` — these are generated by the notebook and are not
committed to the repo (see `.gitignore`).

## Known limitations & next steps

- Artist-level features (`artist_mean_pop`) are computed from the training
  set only (previously this leaked test-set info — see the notebook's
  section 2 for the fix). Unknown/new artists uploaded through the app fall
  back to the training set's global average popularity.
- The app shows a point estimate with an 80% error margin derived from
  test-set residuals, not a full predictive interval.
- No automated tests / CI yet.
- Not yet deployed — see [Streamlit Community Cloud](https://streamlit.io/cloud)
  for a free one-click deploy from this repo.
